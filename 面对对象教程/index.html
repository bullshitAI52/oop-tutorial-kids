📝 第一章：认识「类」与「对象」

🏗️ 类 (Class)：就像一份神奇的「设计图纸」或「饼干模具」。它描述了将要做出来的东西有哪些特征，以及能做什么动作，但图纸本身不是真实的猫，只是设计方案。

🐱 对象 (Object)：就是用图纸真实造出来的「实物」！用猫的图纸，可以造出好多只真实的猫：黑猫汤姆、白猫泡面……它们每一只都是一个「对象」。

🍪 换个例子更好懂：「饼干模具」就是类，决定了饼干的形状；用模具压出来的每一片饼干就是对象。同一个模具能压出好多片饼干（很多对象），每片可以有不同口味（属性值不同）。

🏷️ 第二章：属性 和 方法

每一只猫都有自己的「特征信息」（属性）和会做的「动作」（方法）！

🏷️ 属性 (Attribute) = 对象的「特征/信息」

属性就是用来描述对象长什么样、有什么特点的信息。就像学校每位同学都有自己的「档案」，里面写着姓名、年龄、身高……这些都是属性！

在代码里，我们在 __init__ 方法里用 self.属性名 = 值 给每只猫填写它的「档案」。其中 self 就是「我自己这只猫」的意思。

🐱
汤姆的属性档案

📌 name（名字）= "汤姆"
🎨 color（颜色）= "黑色"
🎂 age（年龄）= 2

🐈
泡面的属性档案

📌 name（名字）= "泡面"
🎨 color（颜色）= "白色"
🎂 age（年龄）= 3

⚡ 方法 (Method) = 对象能做的「动作/行为」

方法就像猫的「才艺」，是对象能做的事情。猫会「喵喵叫」、会「跑步」、会「睡觉」，这些都是方法！

在代码里，方法用 def 动作名(self): 来定义。注意写上 self，这让方法知道是「哪只猫」在执行动作。

😺
meow()
喵喵叫的动作
🏃
run()
跑步的动作
😴
sleep()
睡觉的动作

💻 Python 语法：属性和方法怎么写？怎么调用？

# ====== 第一步：定义类（图纸）======
class Cat:

    def __init__(self, name, color, age):  # 造猫时自动调用，给每只猫填写档案
        self.name  = name    # ← 属性：名字
        self.color = color   # ← 属性：颜色
        self.age   = age     # ← 属性：年龄

    def meow(self):                # ← 方法：喵喵叫动作
        print(f"{self.name} 说：喵喵喵～我是{self.color}猫！")

    def run(self):                 # ← 方法：跑步动作
        print(f"{self.name} 飞速奔跑啦！🏃")

# ====== 第二步：用图纸造猫（创建对象）======
cat1 = Cat("汤姆", "黑色", 2)   # 造出黑猫汤姆
cat2 = Cat("泡面", "白色", 3)   # 造出白猫泡面

# ====== 读取属性 → 对象.属性名（不加括号）======
print(cat1.name)    # 输出：汤姆
print(cat2.color)   # 输出：白色
print(cat1.age)     # 输出：2

# ====== 调用方法 → 对象.方法名()（要加括号！）======
cat1.meow()   # 输出：汤姆 说：喵喵喵～我是黑色猫！
cat2.meow()   # 输出：泡面 说：喵喵喵～我是白色猫！
cat1.run()    # 输出：汤姆 飞速奔跑啦！🏃
🔑 快速记忆口诀：
🏷️ 属性 = 猫「是什么、有什么」，读取用 对象.属性名（不加括号）
⚡ 方法 = 猫「能做什么」，调用用 对象.方法名()（记得加括号 ()！）

🎁 第一法宝：封装 (Encapsulation)

把复杂的危险零件藏起来，只留给你好玩的按钮！

💡 图解故事：就像一台游戏机🕹️。游戏机里面有成千上万根复杂的电线和容易弄坏的芯片，这些就是复杂的代码，被一层漂亮坚固的塑料外壳“封装”藏起来了。
你在外面根本看不到电线，你只需要按手柄上的“跳跃”按键（留出来的方法），就能直接玩游戏啦！这就叫“封装”！

💻 Python 语法：怎么写“封装”？

# 就像设计一个游戏机类
class GameConsole:
    def __init__(self):
        # 把复杂的电线藏起来 (变量名前加下划线 _ 代表隐藏)
        self._wire_count = 1000
        
    def jump_button(self): # 只留出简单的相关操作方法
        print("🎮 游戏角色跳起来啦！")

# 👇 怎么调用？很简单！
my_console = GameConsole()    # 买一台新游戏机 (创建对象)
my_console.jump_button()      # 直接按按钮，根本不需要管里面的电线！

👩‍👦 第二法宝：继承 (Inheritance)

子类可以自动拥有父类的能力，还能自己学会新绝招！

💡 图解故事：有一只猫妈妈（我们叫它父类），她天生就会“喵喵叫”和“跑”。
猫妈妈生了一只猫宝宝（我们叫它子类）。猫宝宝一出生，就继承了妈妈所有的技能，不用学就会“喵喵叫”和“跑”。而且，聪明的小猫宝宝还自己发明了新绝招：“后空翻”！

🐱

拥有技能: 喵喵叫() 跑步()

⬇️ 遗传 / 学会啦 ⬇️
😸😼

猫宝宝们 (子类)

继承的技能: 喵喵叫() 跑步()

🔥 自己领悟的新绝招: 后空翻()

💻 Python 语法：怎么写“继承”？

# 第一步：定义父类（猫妈妈）
class CatMom:
    def meow(self):
        print("喵喵！我是猫妈妈。")

# 第二步：定义子类，括号里写上猫妈妈的名字，表示继承她！
class CatBaby(CatMom): 
    def backflip(self):  # 子类自己学会的新绝招
        print("✨ 看我的后空翻！")

# 👇 怎么调用？
baby = CatBaby()      # 诞生一只猫宝宝 (创建对象)
baby.meow()           # 调用父类的本领 (输出: 喵喵！我是猫妈妈。)
baby.backflip()       # 调用自己的新绝招 (输出: ✨ 看我的后空翻！)

🎭 第三法宝：多态 (Polymorphism)

听到完全一样的口令，不同的对象有不同的反应！

💡 图解故事：动物园园长（你！）对着所有的动物大喊同一个指令功能：
“大家叫一声()！”
结果，小狗汪汪叫，小鸭子嘎嘎叫，小猫喵喵叫。同一个指令，大家却用自己独有的方式去执行，这就叫“多态”！

💻 Python 语法：怎么写“多态”？

# 大家都有相同的方法名 make_sound()，但里面实现的事情不一样
class Dog:
    def make_sound(self):
        print("🐶 汪汪汪！")

class Duck:
    def make_sound(self):
        print("🦆 嘎嘎嘎！")

# 👇 怎么调用？园长只需写一个发令方法，传什么进来就让什么叫
def animal_shout(animal):
    animal.make_sound()  # 统一发出“叫一声”的口令

# 传入不同对象，反应不同！这就叫多态！
animal_shout(Dog())   # 输出: 🐶 汪汪汪！
animal_shout(Duck())  # 输出: 🦆 嘎嘎嘎！
